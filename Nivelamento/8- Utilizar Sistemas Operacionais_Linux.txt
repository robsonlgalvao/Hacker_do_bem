O Linux é um dos sistemas operacionais de código aberto, ou seja, seu código-fonte está disponível para qualquer pessoa modificar, melhorar e distribuir de acordo com os termos das licenças de software livre. 

Desenvolvido pelo engenheiro de software finlandês Linus Torvalds, no início do anos 1990, os Linux cresceu rapidamente em popularidade devido a facilidade no processo de colaboração no desenvolvimento do sistema operacional. 

Linux é o kernel do sistema operacional

Sistemas de arquivos

EXT4
Integridade: Journaling/FCSK
criptografia: sim
compactação: não
permissão: sim
Limites: Arquivos 16TB Partição 1EB
Outros Recursos: Compatibilidade Windows MacOs, Cotas em disco,
Estabilidade e velocidade

BTRFS(B-Tree file system)
Integridade:	Copia em gravação(COW)/Auto reparo
Criptografia:	Sim
Compactação:	Sim
Snapshot:	Sim
Permissão:	Sim
Limites:	Arquivos 16EB/Partição 16EB
Outros Recursos: Redimecionamento de volume, RAID, Backup incremental, Desduplicação dedados

JFS - Journaled File System
Integridade 	- Journaling
Criptografia:	- Não
Compactação:	- Não
Snapshots:	- Não 
Permissões	- Sim
Limite: 	- arquivo 4PB/partição 32PB

XFS - High Performance Scalable File System
Integridade:	Journaling
Compactação:	Não
Criptografia:	Não
Snapshots:	Não
Permissões:	Sim
Limites:	Arquivos de 8EB/Partição 16EB
Outros Recursos: Aumentar o tamanho, cotas, leitura e gravação paralela.

ReiserFS
Integridade:	Journaling/FSCK
Criptografia:	Não
Compactação:	Não
Snapshot:	Não
Permissões	Não
Permissões:	Sim
Limites:	Arquivos 8TB/Partição 16TB

ZFS (criado pela oracle)
Integridade:	CoW Checksum/auto reparo
Criptografia:	Sim
Compactação:	Sim
Snapshots:	Sim
Permissões:	Sim
Limites:	Arquivo de 16EB/Partição 256 quatrilhões de Zettabyte
Outros Recursos RAID-Z, Desduplicação de dados

Permissões

leitura	-([R]ead) - Consegue ler arquivo/pasta
escrita	-([W]rite) - Consegue alterar/apagar arquivo/pasta
execução-(e[X]ecute)-Consegue executar o arquivo/acessar pasta

podem ser do:

Usuário proprietário[user owner](u)
Grupo[group](g)
Outros[other](o)

ls -l - listar diretorios/ arquivos com detalhes

d  (diretório)
-  (Arquivo)
l  (link)

ex: drwxrwxr-x	(d rwx rwx r-x)

são 10 marcações 
1ª(se é pasta/diretório,arquivo ou link); 
2ª,3ª e 4ª(usuário);
5ª,6ª,7ª(grupo); 
8ª,9ª e 10ª(outros usuários);

tabela de permissões

decimal		binario		permissão

0		000		sem permissão
1		001		Executar 		(--x)
2		010		escrita/write		(-x-)
3		011		executar e escrita	(-wx)
4		100		Leitura/read		(r--)
5		101		Leitura e execução	(r-x)
6		110		leitura e escrita	(rw-)
7		111		leitura, escrita e execução(rwx)

drwxrwxr-x = d 775 ou d 111 111 101


Alterar permissões de pasta

ls -la (exibe arquivos/pastas ocultos)

ch mod (usado para mudar permissões)
ex:(chmod u+rwx g-rwx o+rwx) ou (chmod 707)
nesse exemplo colocamos(+) no usuário(u) leitura(r) escrita(w) e execusão(x); 
já no grupo retiramos(-), leitura(r) escrita(w) e execusão(x)
e no outros usuários colocamos(+), leitura(r) escrita(w) e execusão(x)

ex total: chmod 707 ou u+rwx g-rwx o+rwx caminho pasta desejada.

OBS: o padrão é 775


chown	- muda o proprietário. ex: sudo crown novoProprietário nomePasta/arquivo

chgrp	- Muda o grupo. ex: sudo chgrp grupoDestino nomePasta/arquivo.
